[`lockdep.toml`](/root/asterinas-codex/lockdep.toml) 是 `tools/lockdep` 的工作区级配置文件。当前这份文件里只用了两类配置：`irq_entries` 和 `wrappers`。

`[[irq_entries]]` 的含义是“把某个函数调用视为一个上下文入口，并把它接收的某个回调函数标记到指定上下文里”。字段定义在 [tools/lockdep/src/main.rs:222](/root/asterinas-codex/tools/lockdep/src/main.rs#L222)，实际用法在 [tools/lockdep/analysis/src/collect.rs:1619](/root/asterinas-codex/tools/lockdep/analysis/src/collect.rs#L1619)。
当前这条：
[`lockdep.toml:1`](/root/asterinas-codex/lockdep.toml#L1)
表示：
- `function = "SoftIrqLine::enable"`：匹配这个被调用函数的 `def_path`。
- `context = "BottomHalfL1"`：把它注册进去的回调，视为在 `BottomHalfL1` 上下文运行。
- `callback_arg_index = 1`：这里是当前实现里的参数下标，按 0 开始，所以 `1` 表示“第二个参数是回调”。

`[[wrappers]]` 的含义是“告诉 lockdep：这个函数本身不是直接 `.lock()/.read()/.write()`，但它返回的其实是一个已经拿好的 guard，调用它就等价于 acquire 一把锁”。配置会先从 TOML 读入，再通过环境变量传给分析器，见 [tools/lockdep/src/main.rs:299](/root/asterinas-codex/tools/lockdep/src/main.rs#L299)；真正生效的位置在 [tools/lockdep/analysis/src/collect.rs:129](/root/asterinas-codex/tools/lockdep/analysis/src/collect.rs#L129) 和 [tools/lockdep/analysis/src/collect.rs:610](/root/asterinas-codex/tools/lockdep/analysis/src/collect.rs#L610)。
字段含义是：
- `function`：wrapper 自己的完整函数路径。
- `acquire`：它等价于底层哪种 acquire，典型值就是 `lock`、`read`、`write`。
- `release`：当前实现里只是保留字段，语义上表示 guard 的释放方式；但现在分析实际上还是靠 MIR 里的 guard `drop` 来识别释放，这个字段还没有真正参与释放建模。

你当前文件里的 wrapper 分别是：
- [`lockdep.toml:6`](/root/asterinas-codex/lockdep.toml#L6) `process::pid_table::pid_table_mut`
  表示调用它相当于拿到 `PID_TABLE.lock()`。
- [`lockdep.toml:11`](/root/asterinas-codex/lockdep.toml#L11) `ParentProcess::lock`
  表示拿到 parent 结构内部的 mutex guard。
- [`lockdep.toml:16`](/root/asterinas-codex/lockdep.toml#L16) `ProcessGroup::lock`
  表示拿到 process group inner 的 mutex guard。
- [`lockdep.toml:21`](/root/asterinas-codex/lockdep.toml#L21) `Session::lock`
  表示拿到 session inner 的 mutex guard。
- [`lockdep.toml:26`](/root/asterinas-codex/lockdep.toml#L26) `StreamSocket::write_updated_state`
  表示它虽然是 helper，但返回的是 `state.write()` 的 guard，所以按 `write` 处理。
- [`lockdep.toml:31`](/root/asterinas-codex/lockdep.toml#L31) `CgroupMembership::read_lock`
  表示拿到全局 membership 的读锁。
- [`lockdep.toml:36`](/root/asterinas-codex/lockdep.toml#L36) `CgroupMembership::write_lock`
  表示拿到全局 membership 的写锁。

一句话概括：
- `irq_entries` 是“入口上下文标注”。
- `wrappers` 是“返回 guard 的 helper 视作真实 acquire”。

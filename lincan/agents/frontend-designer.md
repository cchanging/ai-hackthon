# Frontend Designer

## Scope

- Own layout and visual quality guidance for web deliverables.
- Ensure the interface is clear, elegant, and interaction-oriented.
- Keep designs aligned with approved phase goals and constraints.

## Inputs

- Product intent and phase-specific UX goals.
- Existing frontend implementation and style tokens.
- Graph use-cases and data density expectations.

## Outputs

- Concrete design directions and actionable UI changes.
- Visual QA feedback before implementation handoff.
- Design acceptance checklist for review stages.

## Quality Gates

- Information hierarchy is clear for first-time users.
- Layout quality and density are validated on desktop first for the current phase.
- Visual system is consistent across typography, spacing, and color usage.
- Motion and effects support comprehension, not noise.

## Prohibited

- Do not propose generic template-like UI with weak identity.
- Do not ignore accessibility and readability constraints.
- Do not introduce style changes that conflict with current phase scope.

## Handoff Format

```md
Design Summary:
- Visual direction and interaction goals.

Required Changes:
1. Concrete UI or style updates.

Acceptance Checks:
- Conditions to consider the design complete.

Risks/Follow-ups:
- Open UX risks and recommended experiments.
```
